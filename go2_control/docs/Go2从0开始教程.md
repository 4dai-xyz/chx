## 3. 核心概念

### WSL

WSL 是 Windows 里的 Linux。你的 Ubuntu 跑在 WSL2 中。ROS2、MuJoCo、Isaac 都在 WSL 里配置。

### ROS2

ROS2 是机器人开发框架，常用概念：

```text
node：一个机器人程序
topic：发布/订阅的数据流
message：topic 中的数据类型
service：请求/响应式接口
```

### DDS / CycloneDDS

DDS 是底层通信机制。Unitree SDK2 和 ROS2 都会用到 DDS。

仿真常用：

```text
Domain ID：1
Interface：lo
```

实机常用：

```text
Domain ID：0
Interface：eth0 或实际连接 Go2 的网卡
```

### SDK2

SDK2 是 Unitree Go2 常用开发接口。你可以用 C++ 或 Python：

```text
读取状态
调用服务
发送运动命令
发送底层电机命令
```

### MuJoCo

MuJoCo 是物理仿真器。它读取 XML 模型，计算刚体、关节、碰撞和接触。

### Isaac Lab

Isaac Lab 是强化学习机器人任务框架。它负责：

```text
机器人配置
场景配置
观测空间
动作空间
奖励函数
终止条件
随机化
训练/播放入口
```

## 7. 运行 Isaac 仿真

### 重要限制：WSL2 里不能按正常方式显示 Isaac 窗口

当前机器的 Ubuntu 是 WSL2。NVIDIA 官方文档明确说明，`isaacsim` Python/Kit 程序不支持在 WSL2 中运行。

这件事很容易被下面的检查结果误导：

```text
torch.cuda.is_available() == True
NVIDIA GeForce RTX 4050 Laptop GPU
isaacsim import ok
```

它们只说明 PyTorch CUDA 计算可用、Isaac Sim Python 包可以导入、Go2 task 可以注册；它们不说明 Isaac 的 RTX/Vulkan 渲染器能在 WSL2 中创建窗口。

WSL2 中常见的实际日志是：

```text
Driver Version: 0.00
Graphics API: Vulkan
No device could be created
Completed setting up the environment
```

最后一行表示物理环境已经创建，程序随后进入持续仿真循环；由于 RTX/Vulkan 设备创建失败，不会出现窗口，继续等待也不会恢复。

当前推荐选择：

```text
1. 在当前 WSL2 中用 MuJoCo 做实时可视化、控制和基础导航验证。
2. 在 WSL2 中实验 Isaac Lab 的 Newton + Viser 浏览器可视化；它不是 RTX 窗口，只能作为几何实时观察方案。
3. 要使用 Isaac 标准实时窗口、RTX 相机、RTX LiDAR、高真实感室内场景，在原生 Windows 11 或原生 Ubuntu 安装 Isaac Sim/Lab。
4. 也可以使用满足配置的远程 Linux GPU 工作站运行 Isaac。
5. 原生 Windows Isaac Sim 可以通过 ROS2 Bridge 与 WSL2 中的 ROS2 节点通信。
```

另外，当前 RTX 4050 Laptop 只有 6 GB 显存，低于当前 Isaac Lab 官方建议的 16 GB 显存。即使迁移到原生系统，也应从单环境、小场景开始，并准备遇到显存不足。

### 第 2 步：WSL2 中尝试 Isaac Lab 浏览器实时画面

这一步是实验方案，目的是绕开 Isaac Sim/Kit 的 RTX/Vulkan 窗口，改用 Isaac Lab 的 Newton/MJWarp 物理后端和 Viser 浏览器可视化。

先用你自己的 WSL 终端运行：

```bash
cd /home/ros/unitree_dev
ISAAC_DEVICE=cuda:0 NUM_ENVS=1 bash scripts/run_isaaclab_go2_viser.sh
```

这条命令默认运行 `Isaac-Velocity-Flat-Unitree-Go2-Play-v0`。它每一步给动作空间发送 12 个 0：

```text
actions = [0, 0, ..., 0]
```

在 Isaac Lab 的 Go2 速度任务里，动作不是“前进/后退”这种高层速度指令，而是 12 个关节位置目标偏移。0 动作表示维持默认关节目标，不是训练好的平衡/行走策略。因此它只能用于确认模型、物理和浏览器可视化链路；机器人可能摇晃、跪下或倒下，这属于没有策略控制时的正常现象。

启动成功后，终端会打印 Viser 地址。通常可以直接在 Windows 浏览器打开：

```text
http://localhost:8080
```

如果 CUDA 模式卡住或报 Warp/CUDA 驱动错误，可以退一步用 CPU 做确认：

```bash
cd /home/ros/unitree_dev
ISAAC_DEVICE=cpu NUM_ENVS=1 bash scripts/run_isaaclab_go2_viser.sh
```

判断标准：

```text
能打开 http://localhost:8080 并看到 Go2 几何模型运动 -> WSL 浏览器实时观察可用。
3-5 分钟内没有 URL、浏览器打不开、终端没有继续输出 -> 当前机器不适合走这个实验方案，直接用 MuJoCo 或迁移到原生系统。
```

如果你之前看到过下面这个错误：

```text
FileNotFoundError: USD file not found at path:
/home/ros/isaac_go2/assets_cache/Assets/Isaac/6.0/Isaac/Props/UIElements/arrow_x.usd
```

它不是 Go2 模型缺失，而是速度命令的调试箭头 marker 需要额外 USD 资源。当前 `scripts/isaaclab_go2_viser.py` 已经在浏览器模式中自动关闭这些 Kit 调试 marker；重新运行上面的 `run_isaaclab_go2_viser.sh` 即可，不需要手动下载 `arrow_x.usd`。

注意：Viser 不是照片级 Isaac Sim 渲染。它适合观察机器人姿态、关节和轨迹，不适合做 RTX 相机、光照、材质、语义分割、深度图等高真实感传感器实验。

## 10. 工作原理简图

MuJoCo 仿真：

```text
你的 Python/C++ 程序
  -> SDK2 / DDS Domain 1 / lo
  -> unitree_mujoco
  -> MuJoCo Go2 模型
  -> 状态通过 DDS 返回
```

Go2 实机：

```text
你的 Python/C++/ROS2 程序
  -> SDK2/ROS2 / DDS Domain 0 / eth0
  -> Go2 192.168.123.18
  -> 真实机器人
```

Isaac 训练：

```text
Isaac Lab task
  -> 创建多个 Go2 仿真环境
  -> 策略网络输出动作
  -> PhysX 推动物理
  -> reward 反馈给 RSL-RL
  -> PPO 更新策略
```

ORB-SLAM3 / 导航后续联动：

```text
相机/深度/里程计
  -> ORB-SLAM3 或 SLAM 节点
  -> 位姿/地图
  -> 导航或控制节点
  -> Go2 SDK2/ROS2 控制接口
```

## 12. 初学者安全原则

```text
先仿真，后实机。
先只读，后控制。
先高层，后低层。
先小速度，后复杂动作。
先确认急停，再运行任何会让机器狗动的程序。
```

真实 Go2 不是仿真窗口，任何控制命令都可能让它移动、摔倒或碰到人。第一次实机测试，只读状态就已经足够。

## 13. MuJoCo 窗口不显示怎么办

如果运行：

```bash
cd /home/ros/unitree_dev
bash scripts/run_mujoco_python.sh
```

终端里有 DDS 或 pygame 输出，另一个终端通信也正常，但窗口不出现，优先按下面顺序检查。

### 13.1 确认 Python MuJoCo 版本

当前推荐版本是：

```text
mujoco 3.3.6
```

检查：

```bash
/home/ros/unitree_dev/.venv-unitree/bin/python -c 'import mujoco; print(mujoco.__version__)'
```

如果不是 `3.3.6`，重新安装：

```bash
/home/ros/unitree_dev/.venv-unitree/bin/python -m pip install --upgrade --force-reinstall 'mujoco==3.3.6'
```

本工作区的 `scripts/setup_python_env.sh` 已经固定安装 `mujoco==3.3.6`。

### 13.2 检查 WSLg/OpenGL

```bash
echo $DISPLAY
echo $WAYLAND_DISPLAY
glxinfo -B
```

能看到 OpenGL renderer 信息，说明 WSL 图形链路基本正常。

也可以运行：

```bash
glxgears
```

如果 `glxgears` 都没有窗口，问题在 WSLg/图形显示，不在 Go2 仿真。

### 13.3 使用 C++ 版作为备用

Python viewer 如果还有问题，可以先运行 C++ 版：

```bash
cd /home/ros/unitree_dev
bash scripts/run_mujoco_cpp.sh
```

C++ 版是官方 README 中推荐的仿真器路线。本工作区已修复 C++ 运行脚本，使它能正确找到 `simulate/config.yaml` 和 `unitree_robots/go2/scene.xml`。
