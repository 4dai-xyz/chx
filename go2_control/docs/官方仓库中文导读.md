# 官方仓库中文导读

本文件帮助你阅读 `src/` 下的官方仓库。

## 1. unitree_sdk2

路径：

```text
src/unitree_sdk2
```

作用：

- C++ 版 SDK2。
- 可直接通过 DDS 与 Go2 或 MuJoCo 仿真器通信。
- 包含 Go2/B2/G1/H1/A2 等多种机器人示例。

Go2 重点示例：

```text
example/go2/go2_robot_state_client.cpp
example/go2/go2_sport_client.cpp
example/go2/go2_stand_example.cpp
example/go2/go2_low_level.cpp
```

安全提醒：

- `go2_low_level.cpp` 是底层电机控制，风险高。
- `go2_sport_client.cpp` 会发运动命令。
- `go2_stand_example.cpp` 会发站立/趴下动作命令。
- `go2_robot_state_client.cpp` 会切换 `sport_mode` 服务，并非纯读取。

优先学习：

```text
include/unitree/robot/channel
include/unitree/idl/go2
example/helloworld
example/go2
```

## 2. unitree_ros2

路径：

```text
src/unitree_ros2
```

作用：

- 提供 Go2 的 ROS2 消息定义。
- 让你可以用 `ros2 topic list`、`ros2 topic echo` 和 ROS2 节点访问 Go2 数据。

Go2 消息路径：

```text
src/unitree_ros2/cyclonedds_ws/src/unitree/unitree_go/msg
```

重要消息：

```text
SportModeState.msg
LowState.msg
LowCmd.msg
WirelessController.msg
IMUState.msg
MotorState.msg
```

优先运行只读示例：

```text
src/unitree_ros2/example/src/src/read_motion_state.cpp
src/unitree_ros2/example/src/src/read_low_state.cpp
src/unitree_ros2/example/src/src/read_wireless_controller.cpp
```

## 3. unitree_mujoco

路径：

```text
src/unitree_mujoco
```

作用：

- Unitree 官方 MuJoCo 仿真器。
- 支持 Go2 模型和 sim-to-real 示例。
- 当前版本主要支持底层控制开发和验证。

关键目录：

```text
simulate/          C++ 仿真器，官方推荐
simulate_python/   Python 仿真器，上手更快
unitree_robots/    Go2 等机器人 MJCF 模型
example/           C++/Python/ROS2 sim-to-real 示例
terrain_tool/      地形生成工具
```

Go2 模型：

```text
unitree_robots/go2/go2.xml
unitree_robots/go2/scene.xml
unitree_robots/go2/scene_terrain.xml
```

仿真默认通信：

```text
Domain ID: 1
Interface: lo
```

实机默认通信：

```text
Domain ID: 0
Interface: 连接 Go2 的网卡，例如 eth0
```

## 4. unitree_sdk2_python

路径：

```text
src/unitree_sdk2_python
```

作用：

- Python 版 SDK2。
- `unitree_mujoco/simulate_python` 和 Python 示例会使用它。

当前已安装到：

```text
/home/ros/unitree_dev/.venv-unitree
```

验证：

```bash
cd /home/ros/unitree_dev
env -u LD_LIBRARY_PATH -u PYTHONPATH -u CONDA_PREFIX -u CONDA_DEFAULT_ENV \
  .venv-unitree/bin/python -c 'import unitree_sdk2py, cyclonedds, mujoco, pygame; print("ok")'
```

优先看：

```text
unitree_sdk2py/core/channel.py
unitree_sdk2py/idl/unitree_go
example/helloworld
example/wireless_controller
```

## 5. 视觉定位 / SLAM 工作区

路径：

```text
/home/ros/ros2_orbslam3
```

这不是 `unitree_dev/src/` 下面的官方 Go2 仓库，但它是你的“仿真、定位、导航”项目里非常关键的一条线。它应该保持独立环境，后续通过 ROS2 topic 和 TF 接入 Go2 仿真。

关键目录：

```text
Opensource code/ORB_SLAM3-master   ORB-SLAM3 原始源码
Opensource code/DPVO-main          DPVO 原始源码
Opensource code/vggt-main          VGGT 原始源码
src/orbslam3_wrapper               ORB-SLAM3 的 ROS2 wrapper
src/video_publisher                本地视频转 ROS2 Image
src/dpvo_localization              DPVO 离线轨迹和增强分析
src/kv_tracker                     基于 VGGT 的 KV-Tracker 复现
launch/orbslam3_video.launch.py    本地视频 + ORB-SLAM3
launch/dpvo_offline.launch.py      DPVO 离线运行
```

ORB-SLAM3 重点 topic：

```text
/camera/image_raw       输入图像
/yolo/dynamic_mask      动态区域 mask
/slam/pose              相机位姿
/slam/odom              相机里程计
/slam/tracking_state    跟踪状态
/slam/map_points        地图点云
/slam/debug_image       调试图像
```

优先学习：

```text
launch/orbslam3_video.launch.py
src/orbslam3_wrapper/src/mono_node.cpp
src/video_publisher/video_publisher/video_publisher_node.py
launch/dpvo_offline.launch.py
src/dpvo_localization/dpvo_localization/run_dpvo_video.py
src/kv_tracker/README.md
```

## 6. 推荐阅读顺序

1. 先读 `docs/Go2从0开始教程.md`。
2. 再读 `docs/仿真运动控制与具身导航最快入门路线.md`。
3. 只查命令时看 `docs/命令速查.md`。
4. 如果学习 MuJoCo，看 `src/unitree_mujoco/readme_zh.md`，了解仿真结构。
5. 如果学习 ROS2，看 `src/unitree_ros2/README _zh.md`，了解 ROS2 连接方式。
6. 如果学习 SDK2，看 `src/unitree_sdk2/README.md`，了解 SDK2 编译和安装。
7. 如果学习 Isaac，看 `docs/IsaacSimLab_Go2复现指南.md`。
8. 如果学习 ORB-SLAM3 / DPVO / VGGT，看 `/home/ros/ros2_orbslam3` 下的文档和 launch 文件。
9. 最后读源码示例。

当前 Isaac Lab 不在 `src/` 目录下，而是在：

```text
/home/ros/isaac_go2/IsaacLab
```

当前唯一 Isaac conda 环境是：

```text
/home/ros/miniconda3/envs/env_isaaclab312
```

## 7. 不建议初学者先看的内容

```text
底层电机控制
低层 CRC 细节
人形机器人 G1/H1 示例
高频 lowcmd 发布
实机 sport/low-level 控制示例
```

这些内容可以学，但建议在你已经会仿真、会只读 topic、会急停和安全检查之后再碰。
