## 5. Isaac Sim / Isaac Lab 环境

云服务器迁移说明：

```text
详见 docs/Isaac云服务器迁移指南.md
```

检查 Isaac 环境：

```bash
cd /home/ros/unitree_dev

```

平台限制：

```text
当前 Ubuntu 是 WSL2。NVIDIA 官方不支持在 WSL2 中运行 Isaac Sim
Python/Kit 图形程序。PyTorch CUDA 为 True 不等于 RTX/Vulkan 窗口可用。
本机需要稳定实时画面时请优先使用 MuJoCo。
```

WSL 中尝试 Isaac Lab 浏览器实时画面，实验方案：

```bash
cd /home/ros/unitree_dev
ISAAC_DEVICE=cuda:0 NUM_ENVS=1 bash scripts/run_isaaclab_go2_viser.sh
```

默认任务是 `Isaac-Velocity-Flat-Unitree-Go2-Play-v0`，脚本发送的是 12 维零关节动作，只用于看模型和可视化链路，不是训练好的行走控制。

打开：

```text
http://localhost:8080
```

如果 3-5 分钟内没有 URL 或页面没有响应，按 `Ctrl+C` 结束，改用 MuJoCo 或原生系统 Isaac。

如果曾经报 `Isaac/Props/UIElements/arrow_x.usd` 缺失，直接重新运行上面的命令。当前脚本已经关闭了会触发这个资源的速度调试箭头 marker。

在原生 Windows/Ubuntu 或远程 Linux GPU 机器上，Go2 zero agent 参考命令：

```bash
cd /home/ros/unitree_dev
DISABLE_FABRIC=0 ISAAC_DEVICE=cuda:0 NUM_ENVS=1 TASK=Isaac-Velocity-Flat-Unitree-Go2-v0 bash scripts/isaaclab_go2_play.sh
```

检查 Python/CUDA 环境：

```bash
cd /home/ros/unitree_dev
bash scripts/isaaclab_check.sh
```

在 WSL2 中，即使 `cuda_available` 是 `True`，Isaac 仍可能显示 `Driver Version: 0.00`、`No device could be created`，这是 RTX/Vulkan 图形设备没有创建成功。

## 9. 环境被 conda 污染时

你的 shell 里可能有 DPVO/miniconda 的 `libtinfo.so.6` 警告。通常可以忽略；如果命令异常，用下面方式清理：

```bash
env -u LD_LIBRARY_PATH -u PYTHONPATH -u CONDA_PREFIX -u CONDA_DEFAULT_ENV <你的命令>
```

MuJoCo/ROS2 优先用：

```bash
source /home/ros/unitree_dev/scripts/unitree_env.sh
```

Isaac 优先用：

```bash
cd /home/ros/unitree_dev
bash scripts/isaaclab_check.sh
```
