# ros2_orbslam3 系统运行总说明

本文档是当前工程的总入口，目标是让读者可以直接启动和使用以下模块：

1. ROS2 视频发布节点
2. ORB-SLAM3 在线单目 SLAM
3. DPVO 视频轨迹估计
4. VGGT 多视图几何重建
5. KV-Tracker / KV_Track3r 复现与可视化
6. people_bev_tracker 行人 BEV 定位与二维平面图输出

当前工程根目录：

```bash
/home/ros/ros2_orbslam3
```

当前主要输入视频：

```text
resources/input_video.mp4
```

已经叠加 BEV/分割可视化的视频：

```text
resources/input_video.mp4_bev.mp4
```

除非特别说明，主算法输入优先使用原始视频 `resources/input_video.mp4`，不要把 `_bev.mp4` 当作主输入。

## 1. 总原则：两类环境不要混用

当前工程同时存在 ROS2/C++ 节点和深度学习 Python 环境。最容易出错的是把 DPVO conda 环境的 `PYTHONPATH` 带进 ROS2 节点，导致 ROS Humble 的 `cv_bridge` 加载 NumPy 2.x 后报错：

```text
A module that was compiled using NumPy 1.x cannot be run in NumPy 2.2.6
AttributeError: _ARRAY_API not found
```

因此分成两类终端。

### 1.1 ROS2 / ORB-SLAM3 / video_publisher 终端

用于：

```text
video_publisher
ORB-SLAM3 wrapper
rviz2
rqt_image_view
ros2 topic echo/hz
```

必须使用干净 ROS Python：

```bash
cd /home/ros/ros2_orbslam3

unset PYTHONPATH
export PYTHONNOUSERSITE=1

source /opt/ros/humble/setup.bash
source install/setup.bash
```

建议进一步清理 DPVO conda 的库路径：

```bash
export LD_LIBRARY_PATH="$(printf '%s' "$LD_LIBRARY_PATH" | tr ':' '\n' | grep -v '/home/ros/miniconda3/envs/dpvo' | paste -sd:)"
```

验证 `cv_bridge` 是否正常：

```bash
python3 - <<'PY'
import sys
print("python:", sys.executable)
import numpy
print("numpy:", numpy.__version__, numpy.__file__)
import cv_bridge
print("cv_bridge ok")
PY
```

期望看到系统 NumPy，例如：

```text
python: /usr/bin/python3
numpy: 1.21.5 /usr/lib/python3/dist-packages/numpy/__init__.py
cv_bridge ok
```

如果看到：

```text
/home/ros/miniconda3/envs/dpvo/lib/python3.10/site-packages/numpy
```

说明 ROS 终端环境还不干净，需要重新 `unset PYTHONPATH`。

### 1.2 DPVO / VGGT / KV-Tracker / people_bev_tracker 终端

用于：

```text
DPVO
VGGT
KV-Tracker
people_bev_tracker
YOLO / Ultralytics
Rerun
Viser
```

通常使用 conda 的 `dpvo` 环境：

```bash
cd /home/ros/ros2_orbslam3
conda activate dpvo
```

如果需要 ROS2 命令也可以 `source /opt/ros/humble/setup.bash`，但深度学习脚本本身不依赖 ROS topic。

## 3. ROS2 视频发布节点

视频发布节点负责把 `resources/input_video.mp4` 发布为 ROS topic：

```text
/camera/image_raw
/camera/image_raw_full
```

同时可以从 `resources/input_video.mp4_bev.mp4` 抠黄色/绿色区域，发布动态 mask：

```text
/yolo/dynamic_mask
```

### 3.1 启动视频节点

打开一个干净 ROS 终端：

```bash
cd /home/ros/ros2_orbslam3

unset PYTHONPATH
export PYTHONNOUSERSITE=1

source /opt/ros/humble/setup.bash
source install/setup.bash

ros2 launch ./launch/video_only.launch.py
```

### 3.2 检查图像是否发布

另开一个干净 ROS 终端：

```bash
cd /home/ros/ros2_orbslam3

unset PYTHONPATH
export PYTHONNOUSERSITE=1

source /opt/ros/humble/setup.bash
source install/setup.bash

ros2 topic list
ros2 topic hz /camera/image_raw
```

如果正常，会看到 `/camera/image_raw` 有稳定频率。

### 3.3 用 RViz2 看画面

启动：

```bash
rviz2
```

在 RViz2 中：

```text
Add -> By topic -> /camera/image_raw -> Image
```

也可以添加：

```text
/yolo/dynamic_mask -> Image
```

注意：视频发布节点只负责发布 topic，不会自动弹出画面。必须用 RViz2、`rqt_image_view` 或自定义 viewer 订阅后才能看到图像。

## 4. ORB-SLAM3 在线单目 SLAM

ORB-SLAM3 wrapper 是 ROS2 在线节点，订阅：

```text

/yolo/dynamic_mask
```

```text

/slam/odom
/slam/tracking_state
/slam/map_points
/slam/debug_image
```

### 4.1 启动前设置 ORB-SLAM3 路径

每个 ORB-SLAM3 终端都设置：

```bash
export ROS2_ORBSLAM3_REPO=/home/ros/ros2_orbslam3
export ORB_SLAM3_DIR="/home/ros/ros2_orbslam3/project code/ORB_SLAM3-master"
export ORB_SLAM3_LIBRARY="$ORB_SLAM3_DIR/lib/libORB_SLAM3.so"
export LD_LIBRARY_PATH="$ORB_SLAM3_DIR/lib:$ORB_SLAM3_DIR/Thirdparty/DBoW2/lib:$ORB_SLAM3_DIR/Thirdparty/g2o/lib:$LD_LIBRARY_PATH"
```

## 5. DPVO 视频轨迹估计

当前 DPVO 封装入口是：

```text
dpvo_localization/run_dpvo_video
```

它不是订阅 `/camera/image_raw` 的 ROS 在线节点，而是读取视频文件或图片目录，逐帧运行 DPVO。可以理解为“离线/准在线视频轨迹估计”。

### 5.1 启动 DPVO 推荐命令

使用 conda `dpvo` 环境：

```bash
cd /home/ros/ros2_orbslam3
conda activate dpvo

source /opt/ros/humble/setup.bash
source install/setup.bash

ros2 run dpvo_localization run_dpvo_video \
  --dpvo-root "project code/DPVO" \
  --imagedir "resources/input_video.mp4" \
  --calib "project code/DPVO/calib/custom_mall.txt" \
  --name input_video_clean \
  --stride 2 \
  --save_trajectory \
  --plot
```

### 5.2 开启 DPVO viewer

如果本机显示环境和 Pangolin/OpenGL 正常，可以加 `--viz`：

```bash
ros2 run dpvo_localization run_dpvo_video \
  --dpvo-root "project code/DPVO" \
  --imagedir "resources/input_video.mp4" \
  --calib "project code/DPVO/calib/custom_mall.txt" \
  --name input_video_clean_viz \
  --stride 2 \
  --save_trajectory \
  --plot \
  --viz
```

如果 `--viz` 失败，先去掉 `--viz`，保证轨迹保存成功。

### 5.3 输出位置

DPVO 原始轨迹通常保存到：

```text
project code/DPVO/saved_trajectories/input_video_clean.txt
```

增强后处理输出：

```text
output/dpvo_enhanced/input_video_clean/
```

people BEV 默认读取的 TUM 轨迹是：

```text
output/dpvo/trajectory_tum.txt
```

如果需要手动复制：

```bash
mkdir -p output/dpvo
cp "project code/DPVO/saved_trajectories/input_video_clean.txt" output/dpvo/trajectory_tum.txt
```

### 5.4 关于 `launch/dpvo_offline.launch.py`

当前该 launch 中仍可能写着旧路径：

```text
Opensource code/DPVO-main
```

而实际 DPVO 路径是：

```text
project code/DPVO
```

所以更稳的方式是直接用上面的 `ros2 run dpvo_localization run_dpvo_video ...` 并显式传入路径。

## 6. VGGT 多视图几何重建

VGGT 不是传统实时 VO。它更像多视图几何推理器：输入一组图片，输出相机、深度、置信度和点云。

当前官方代码路径：

```text
project code/VGGT/vggt-main
```

注意：部分脚本默认 VGGT 路径可能还是旧的 `Opensource code/...`，运行时建议显式传：

```bash
--vggt-root "project code/VGGT/vggt-main"
```

### 6.1 VGGT 环境

使用 conda `dpvo` 环境：

```bash
cd /home/ros/ros2_orbslam3
conda activate dpvo
```

如果缺依赖，按官方 `requirements.txt` 或本脚本报错提示安装。不要重复安装已有 PyTorch。

### 6.2 滑动窗口运行 VGGT 视频

适合长视频，显存压力相对小：

```bash
python scripts/run_vggt_video.py \
  --video resources/input_video.mp4 \
  --vggt-root "project code/VGGT/vggt-main" \
  --output-dir output/vggt_input_video \
  --frame-step 30 \
  --window-size 8 \
  --window-stride 4 \
  --preprocess pad \
  --conf-percentile 85 \
  --max-points 60000
```

如果显存不足：

```bash
--window-size 4
--frame-step 45
```

### 6.3 单场景关键帧重建

适合生成“一组相机 + 一张点云”的展示：

```bash
python scripts/run_vggt_scene.py \
  --video resources/input_video.mp4 \
  --vggt-root "project code/VGGT/vggt-main" \
  --output-dir output/vggt_scene_full \
  --num-frames 16 \
  --preprocess pad
```

如果显存不足，降低：

```bash
--num-frames 8
```

### 6.4 打开 VGGT 3D 可视化

如果 `run_vggt_video.py` 已生成结果：

```bash
python scripts/open_vggt_viser.py \
  --result-dir output/vggt_input_video \
  --port 8080 \
  --aligned \
  --max-points 100000
```

浏览器打开：

```text
http://localhost:8080
```

### 6.5 VGGT 输出

常见输出目录：

```text
output/vggt_input_video/
output/vggt_scene_full/
```

常见文件：

```text

summary.json
summary.txt
camera_centers.txt
windows/
aligned_full/
vggt_points.ply
predictions.npz
```

## 8. people_bev_tracker 行人二维 BEV 定位

people BEV 的作用：

```text
原始视频
  -> YOLO-seg + BoT-SORT / ByteTrack
  -> 行人脚底像素
  -> DPVO 相机位姿 + 相机内参 + 地面平面
  -> 行人世界坐标
  -> 二维 BEV 平面图
```

代码路径：

```text
src/people_bev_tracker
```

默认输入：

```text
resources/input_video.mp4
config/KannalaBrandt8_1280x720.yaml
output/dpvo/trajectory_tum.txt
```

### 8.1 people BEV 环境

使用 conda `dpvo` 环境：

```bash
cd /home/ros/ros2_orbslam3
conda activate dpvo
```

安装依赖：

```bash
pip install -r src/people_bev_tracker/requirements.txt
```

如果你已经有官方 Ultralytics 本地库：

```bash
pip install -e thirdparty/official/ultralytics
```

如果没有，也可以使用 pip 安装的 `ultralytics`。

### 8.2 检查视频信息

```bash
python src/people_bev_tracker/scripts/inspect_video.py \
  --video resources/input_video.mp4
```

### 8.3 短测试

先跑 30 帧，确认检测、跟踪、BEV 投影能工作：

```bash
python src/people_bev_tracker/scripts/offline_pipeline.py \
  --video resources/input_video.mp4 \
  --calib config/KannalaBrandt8_1280x720.yaml \
  --pose output/dpvo/trajectory_tum.txt \
  --output-dir output/people_bev_test \
  --max-frames 30
```

输出：

```text
output/people_bev_test/bev_tracking.mp4
output/people_bev_test/debug_overlay.mp4
output/people_bev_test/people_tracks.json
output/people_bev_test/camera_trajectory.json
```

### 8.4 完整运行

```bash
python src/people_bev_tracker/scripts/offline_pipeline.py \
  --video resources/input_video.mp4 \
  --calib config/KannalaBrandt8_1280x720.yaml \
  --pose output/dpvo/trajectory_tum.txt \
  --output-dir output/people_bev
```

输出：

```text
output/people_bev/bev_tracking.mp4
output/people_bev/debug_overlay.mp4
output/people_bev/people_tracks.json
```

### 8.5 不重跑 YOLO，重新渲染 BEV

如果只是调样式、范围或是否显示轨迹，可以用 JSON 重渲染：

```bash
python src/people_bev_tracker/scripts/render_bev_from_json.py \
  --people output/people_bev/people_tracks.json \
  --camera output/people_bev/camera_trajectory.json \
  --output output/people_bev/bev_tracking_rerender.mp4
```

## 10. 常见问题

### 10.1 视频节点 NumPy 报错

现象：

```text
A module that was compiled using NumPy 1.x cannot be run in NumPy 2.2.6
AttributeError: _ARRAY_API not found
```

原因：

```text
ROS 节点加载到了 conda dpvo 环境中的 NumPy 2.x。
```

解决：

```bash
unset PYTHONPATH
export PYTHONNOUSERSITE=1
source /opt/ros/humble/setup.bash
source install/setup.bash
```

### 10.2 ORB-SLAM3 启动后没有画面

原因：

```text
ROS topic 不会自动显示画面。需要 RViz2 或 rqt_image_view 订阅。
```

检查：

```bash
ros2 topic hz /camera/image_raw
```

显示：

```bash
rviz2
```

添加 `/camera/image_raw` 或 `/slam/debug_image`。

### 10.3 `orbslam3_only` 看起来卡住

`orbslam3_only` 只启动 ORB-SLAM3，不启动视频发布节点。需要另一个终端先启动：

```bash
ros2 launch ./launch/video_only.launch.py
```

或者直接使用：

```bash
ros2 launch ./launch/orbslam3_video.launch.py
```

### 10.4 Pangolin viewer 不显示

可能是 WSL/SSH/OpenGL/DISPLAY 问题。可以禁用 viewer，用 RViz2 看 ROS topic：

```bash
ros2 run orbslam3_wrapper mono_node --ros-args \
  -p vocab_path:="$ORB_SLAM3_DIR/Vocabulary/ORBvoc.txt" \
  -p settings_path:="/home/ros/ros2_orbslam3/config/KannalaBrandt8_960x540.yaml" \
  -p enable_viewer:=false \
  -p enable_dynamic_mask:=true
```

### 10.5 DPVO launch 路径不对

如果 `launch/dpvo_offline.launch.py` 报找不到 `Opensource code/DPVO-main`，不要用它，直接用：

```bash
ros2 run dpvo_localization run_dpvo_video \
  --dpvo-root "project code/DPVO" \
  --imagedir "resources/input_video.mp4" \
  --calib "project code/DPVO/calib/custom_mall.txt" \
  --name input_video_clean \
  --stride 2 \
  --save_trajectory \
  --plot
```

### 10.6 VGGT 脚本找不到源码

现象：

```text
找不到 VGGT 源码目录: Opensource code/...
```

解决：显式指定实际路径：

```bash
--vggt-root "project code/VGGT/vggt-main"
```

### 10.7 people BEV 投影位置不合理

可能原因：

```text
DPVO 单目尺度不是真实米制
地面平面假设不对
世界坐标轴和 BEV 轴选择不对
相机内参分辨率不匹配
```

先确认：

```text
output/dpvo/trajectory_tum.txt 是否存在
config/KannalaBrandt8_1280x720.yaml 是否匹配 pipeline 中处理分辨率
src/people_bev_tracker/config/people_bev_tracker.yaml 中 ground/world 配置
```

## 12. 建议的最终应用路线

如果目标是“商场单目视觉导航 + 相机轨迹 + 行人定位 + 二维真实地图投影”，推荐先按这个顺序推进：

1. 用 ORB-SLAM3 在线查看视频流、调相机标定和动态 mask。
2. 用 DPVO 离线生成稳定的相机轨迹 `output/dpvo/trajectory_tum.txt`。
3. 用 people_bev_tracker 把行人轨迹和相机轨迹画到二维 BEV 平面。
4. 用 VGGT 对关键帧/窗口做局部结构重建，辅助理解商场空间结构。
5. 用 KV-Tracker 复现论文方法，导出 `trajectory_tum.txt`、`trajectory.json`、关键帧和置信度，作为 DPVO/ORB-SLAM3 的对照或替代轨迹源。
6. 最后做尺度对齐、地面平面估计和真实商场平面图配准。

当前最直接可看的成果是：

```text
output/people_bev/bev_tracking.mp4
```

它会在二维平面中同时显示：

```text
相机轨迹 + 动态行人位置 + 行人 track_id
```
